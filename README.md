# 🍔 Online Multi-Resturant Delivery System

## 📌 Project Overview

This project presents a comprehensive study and design of an **Online Food Delivery System**. It follows a structured Software Engineering approach, covering Requirements Engineering, UML Modeling, and Project Planning. The system facilitates seamless interactions between **Customers**,  **Restaurants**, and **Delivery Personnel** with integrated real-time tracking and secure payment processing.

## 🔗 Model Connections (How they integrate)

To maintain system consistency, all diagrams are interconnected:

-   The **Use Case Diagram** defines the high-level interactions between actors.
    
-   These interactions are expanded into step-by-step logic in the **Activity Diagram**.
    
-   The **Context Diagram** identifies the data boundaries between these actors and the system.
    
-   The **State Machine Diagram** tracks the internal status of the "Order" object as it moves through the flows defined in the Activity Diagram.
    
-   The **Gantt & PERT Charts** manage the development lifecycle of all the above models.
